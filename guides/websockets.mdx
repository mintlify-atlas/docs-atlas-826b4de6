The GRVT WebSocket client provides real-time streaming of market data, order updates, position changes, and account events. This guide covers connection management, subscription handling, and data streaming.

## Initialization

Create a WebSocket connection with automatic reconnection:

```typescript
import { WS } from '@grvt/client';

const ws = new WS({
  url: 'wss://ws.grvt.io',
  protocols: [], // optional
  timeout: 30000, // subscription timeout in ms (default: 30000)
  reconnectStrategy: (retries) => {
    // Exponential backoff: 1s, 2s, 4s, 8s, max 30s
    const delay = Math.min(1000 * Math.pow(2, retries), 30000);
    return delay;
  }
});
```

<Note>
The WebSocket client automatically handles reconnection and re-subscription when the connection is lost.
</Note>

## Connection Management

### Connecting

```typescript
// Connect to WebSocket
ws.connect();

// Wait for connection to be ready
await ws.ready();
console.log('WebSocket connected');
```

### Connection Events

```typescript
// Listen for connection open
ws.onConnect(() => {
  console.log('WebSocket connected');
});

// Listen for connection close
ws.onClose((event) => {
  console.log('WebSocket closed:', event.code, event.reason);
});

// Listen for errors
ws.onError((error) => {
  console.error('WebSocket error:', error);
});
```

### Disconnecting

```typescript
// Disconnect and stop reconnection attempts
ws.disconnect();
```

## Market Data Streams

### Candlestick Data

Stream real-time candlestick updates:

```typescript
import { EStream } from '@grvt/client';

const candleSubscription = ws.subscribe({
  stream: EStream.CANDLE,
  params: {
    instrument: 'BTC_USDT_Perp',
    interval: '1H', // '1M', '5M', '15M', '1H', '4H', '1D'
    type: 'MARK' // 'MARK', 'INDEX', 'LAST'
  },
  onData: (candle) => {
    console.log('Candle update:', candle);
    // { timestamp, open, high, low, close, volume }
  },
  onError: (error) => {
    console.error('Candle subscription error:', error);
  }
});
```

## Trading Account Streams

<Note>
Account streams require authentication. Set query parameters with account credentials before subscribing.
</Note>

### Order Updates

```typescript
// Set authentication parameters
ws.setParams({
  api_key: 'your_api_key',
  signature: 'your_signature',
  timestamp: Date.now().toString()
});

const orders = ws.subscribe({
  stream: EStream.ORDER,
```

## Subscription Management

### Multiple Subscriptions

Subscribe to multiple streams at once:

```typescript
const subscriptionIds = ws.subscribes(
  {
    stream: EStream.TICKER_SNAP,
    params: { instrument: 'BTC_USDT_Perp', rate: 1000 },
    onData: (ticker) => console.log('BTC ticker:', ticker)
  },
  {
    stream: EStream.TICKER_SNAP,
    params: { instrument: 'ETH_USDT_Perp', rate: 1000 },
    onData: (ticker) => console.log('ETH ticker:', ticker)
  },
  {
    stream: EStream.TRADE,
    params: { instrument: 'BTC_USDT_Perp' },
    onData: (trade) => console.log('BTC trade:', trade)
  }
);

console.log('Subscription IDs:', subscriptionIds);
```

### Unsubscribing

Unsubscribe from specific streams:

```typescript
// Unsubscribe using subscription ID
ws.unsubscribe(candleSubscription);

// Unsubscribe from multiple streams
ws.unsubscribe(...subscriptionIds);
```

## Complete Trading Example

Here's a complete example monitoring orders and positions:

```typescript
import { WS, EStream } from '@grvt/client';

const ws = new WS({
  url: 'wss://ws.grvt.io',
  reconnectStrategy: (retries) => Math.min(1000 * Math.pow(2, retries), 30000)
});

// Set authentication
ws.setParams({
  api_key: 'your_api_key',
  signature: 'your_signature',
  timestamp: Date.now().toString()
});

// Track application state
const state = {
  positions: new Map(),
  orders: new Map()
};

// Connect and subscribe
ws.onConnect(() => {
  console.log('Connected to WebSocket');
  
  // Subscribe to multiple streams
  ws.subscribes(
    // Position updates
    {
      stream: EStream.POSITION,
      params: { sub_account_id: 'sub_account_123' },
      onData: (position) => {
        state.positions.set(position.instrument, position);
        console.log('Position updated:', position);
      }
    },
    
    // Order updates
    {
      stream: EStream.ORDER,
      params: { sub_account_id: 'sub_account_123' },
      onData: (order) => {
        state.orders.set(order.order_id, order);
        console.log('Order updated:', order);
        
        if (order.status === 'FILLED') {
          console.log('Order filled!', order.order_id);
        }
      }
    },
    
    // Fill notifications
    {
      stream: EStream.FILL,
      params: { sub_account_id: 'sub_account_123' },
      onData: (fill) => {
        console.log('Trade executed:', {
          order_id: fill.order_id,
          price: fill.price,
          size: fill.size,
          fee: fill.fee
        });
      }
    },
    
    // Market data for monitoring
    {
      stream: EStream.TICKER_SNAP,
      params: { instrument: 'BTC_USDT_Perp', rate: 1000 },
      onData: (ticker) => {
        console.log('BTC Price:', ticker.last_price);
      }
    }
  );
});

ws.onError((error) => {
  console.error('WebSocket error:', error);
});

ws.onClose((event) => {
  console.log('WebSocket closed, will reconnect...');
});

// Connect
ws.connect();

// Graceful shutdown
process.on('SIGINT', () => {
  console.log('Shutting down...');
  ws.disconnect();
  process.exit(0);
});
```

## Error Handling

Handle WebSocket errors properly:

```typescript
ws.onError((error) => {
  if (error instanceof WSError) {
    console.error('WebSocket API error:', {
      code: error.code,
      status: error.status,
      message: error.message
    });
    
    // Handle specific error codes
    if (error.code === 401) {
      console.error('Authentication failed, refreshing credentials...');
      // Refresh and reconnect
    }
  } else {
    console.error('WebSocket connection error:', error);
  }
});

// Per-subscription error handling
ws.subscribe({
  stream: EStream.TICKER_SNAP,
  params: { instrument: 'BTC_USDT_Perp' },
  onData: (ticker) => console.log(ticker),
  onError: (error) => {
    console.error('Ticker subscription failed:', error.message);
    // Handle subscription-specific errors
  }
});
```

## Best Practices

<AccordionGroup>
  <Accordion title="Connection Management">
    - Always implement a reconnection strategy with exponential backoff
    - Use `onConnect` to initialize subscriptions after reconnection
    - Monitor connection status with `onClose` and `onError` handlers
    - Call `disconnect()` on application shutdown to close cleanly
  </Accordion>
  
  <Accordion title="Subscription Handling">
    - Subscribe to multiple streams in a single `subscribes()` call for efficiency
    - Store subscription IDs to unsubscribe later
    - Use instrument-specific subscriptions to reduce bandwidth
    - Implement per-subscription error handlers for better debugging
  </Accordion>
  
  <Accordion title="Data Processing">
    - Keep `onData` handlers lightweight to avoid blocking the event loop
    - Use async operations carefully to prevent data processing backlog
    - Consider using a message queue for heavy processing
    - Implement state reconciliation after reconnection
  </Accordion>
  
  <Accordion title="Authentication">
    - Refresh authentication parameters before they expire
    - Use `setParams()` to update credentials without reconnecting
    - Handle 401 errors by refreshing credentials and resubscribing
  </Accordion>
</AccordionGroup>
