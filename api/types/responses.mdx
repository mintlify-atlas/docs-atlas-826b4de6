Response types returned by the GRVT SDK for various API operations.

## WebSocket Feed Data Types

### IWSCandlestickFeedDataV1

Candlestick feed data structure.

```typescript
interface IWSCandlestickFeedDataV1 {
  stream?: string
  selector?: string
  sequence_number?: string
  feed?: ICandlestick
  prev_sequence_number?: string
}
```

<ResponseField name="stream" type="string">
  Stream name (e.g., `candle`)
</ResponseField>

<ResponseField name="selector" type="string">
  Primary selector identifying the subscription
</ResponseField>

<ResponseField name="sequence_number" type="string">
  Sequence number for message ordering. If `useGlobalSequenceNumber` is true, returns global sequence number; otherwise returns gateway sequence number.
</ResponseField>

<ResponseField name="feed" type="ICandlestick">
  Candlestick data matching the request filters
</ResponseField>

<ResponseField name="prev_sequence_number" type="string">
  Previous sequence number for ordering
</ResponseField>

## Common Response Patterns

### Sequence Numbers

All feed data includes sequence numbers for message ordering:

- **Gateway Sequence Number**: Increments locally within each stream, resets on gateway restart
- **Global Sequence Number**: Unique across the cluster, computed as `cluster_sequence_number * 10^5 + dedupCounter`

The type of sequence number depends on the `useGlobalSequenceNumber` configuration.

### Delta vs Snapshot Feeds

**Delta Feed Semantics:**

```typescript
// On subscription: full snapshot sent first
{
  stream: 'ticker.d',
  feed: {
    instrument: 'BTC_USDT_Perp',
    last_price: '50000',
    volume_24h: '1000000',
    // ... all fields
  }
}

// Subsequent updates: only changed fields
{
  stream: 'ticker.d',
  feed: {
    last_price: '50100',  // updated
    // other fields omitted = unchanged
  }
}

// Field set to zero
{
  stream: 'ticker.d',
  feed: {
    last_price: '0'  // explicitly zero
  }
}

// Field set to null
{
  stream: 'ticker.d',
  feed: {
    last_price: ''  // null/empty
  }
}
```

**Snapshot Feed Semantics:**

```typescript
// Every update: complete snapshot
{
  stream: 'ticker.s',
  feed: {
    instrument: 'BTC_USDT_Perp',
    last_price: '50100',
    volume_24h: '1000001',
    // ... all fields always included
  }
}
```

### Orderbook Delta Semantics

For orderbook delta feeds (`rpi.book.d`):

```typescript
// Initial snapshot: all levels
{
  feed: {
    bids: [
      { price: '50000', size: '1.5' },
      { price: '49999', size: '2.0' }
    ],
    asks: [
      { price: '50001', size: '1.0' },
      { price: '50002', size: '0.5' }
    ]
  }
}

// Delta update: only changed levels
{
  feed: {
    bids: [
      { price: '50000', size: '2.0' }  // size updated
    ],
    asks: [
      { price: '50001', size: '0' }  // level removed
    ]
  }
}
```

**Level semantics:**
- Level not published = unchanged
- `size: 'X'` = level updated to size X
- `size: '0'` = level removed
- New levels published when price moves

## Error Responses

WebSocket errors use the `WSError` class:

```typescript
class WSError extends Error {
  code?: number
  status?: number
  message: string
}
```

<ResponseField name="code" type="number">
  Error code from server
</ResponseField>

<ResponseField name="status" type="number">
  HTTP-style status code
</ResponseField>

<ResponseField name="message" type="string">
  Error description
</ResponseField>

**Example error handling:**

```typescript
ws.onError((error) => {
  if (error instanceof WSError) {
    console.error(`Error ${error.code}: ${error.message}`)
    
    // Handle specific error codes
    if (error.code === 1003) {
      // Malformed request (automatically ignored by SDK)
    }
  }
})
```
