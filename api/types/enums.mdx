## Order Enums

### EOrderStatus

Order lifecycle states.

```typescript
enum EOrderStatus {
  PENDING = 'PENDING',
  OPEN = 'OPEN',
  FILLED = 'FILLED',
  REJECTED = 'REJECTED',
  CANCELLED = 'CANCELLED'
}
```

<ParamField path="PENDING" type="string">
  Order sent to matching engine, pending transition to open/filled/rejected
</ParamField>

<ParamField path="OPEN" type="string">
  Order actively matching on the matching engine (unfilled or partially filled)
</ParamField>

<ParamField path="FILLED" type="string">
  Order fully filled and closed. Taker orders can transition directly from PENDING to FILLED.
</ParamField>

<ParamField path="REJECTED" type="string">
  Order rejected by matching engine. Once open, an order cannot be rejected.
</ParamField>

<ParamField path="CANCELLED" type="string">
  Order cancelled by user. Before an order is open, it cannot be cancelled.
</ParamField>

### ETimeInForce

Order time-in-force options.

```typescript
enum ETimeInForce {
  GOOD_TILL_TIME = 'GOOD_TILL_TIME',
  ALL_OR_NONE = 'ALL_OR_NONE',
  IMMEDIATE_OR_CANCEL = 'IMMEDIATE_OR_CANCEL',
  FILL_OR_KILL = 'FILL_OR_KILL',
  RETAIL_PRICE_IMPROVEMENT = 'RETAIL_PRICE_IMPROVEMENT'
}
```

<ParamField path="GOOD_TILL_TIME" type="string">
  GTT - Remains open until cancelled or expired
</ParamField>

<ParamField path="ALL_OR_NONE" type="string">
  AON - Fill the whole order or none of it (Block Trades Only)
</ParamField>

<ParamField path="IMMEDIATE_OR_CANCEL" type="string">
  IOC - Fill as much as possible when hitting orderbook, then cancel
</ParamField>

<ParamField path="FILL_OR_KILL" type="string">
  FOK - Both AON and IOC. Fill fully or cancel immediately.
</ParamField>

<ParamField path="RETAIL_PRICE_IMPROVEMENT" type="string">
  RPI - GTT + PostOnly maker order, only takeable by non-algorithmic UI users
</ParamField>

**Matrix:**

|                       | Must Fill All | Can Fill Partial |
| --------------------- | ------------- | ---------------- |
| Must Fill Immediately | FOK           | IOC              |
| Can Fill Till Time    | AON           | GTT              |

## Usage Examples

### Type-Safe Stream Subscriptions

```typescript
import { ECandlestickInterval, ECandlestickType, EStream } from '@grvt/sdk'

ws.subscribe({
  stream: EStream.CANDLE,
  params: {
    instrument: 'BTC_USDT_Perp',
    interval: ECandlestickInterval.CI_1_H,
    type: ECandlestickType.TRADE
  },
  onData: (candle) => {
    console.log(`${candle.open} ${candle.high} ${candle.low} ${candle.close}`)
  }
})
```

### Order Status Checking

```typescript
import { EOrderStatus, EOrderRejectReason } from '@grvt/sdk'

ws.subscribe({
  stream: 'order',
  params: { sub_account_id: 'abc123' },
  onData: (order) => {
    switch (order.status) {
      case EOrderStatus.FILLED:
        console.log('Order filled!')
        break
      case EOrderStatus.REJECTED:
        console.error('Order rejected:', order.reject_reason)
        if (order.reject_reason === EOrderRejectReason.BELOW_MARGIN) {
          console.error('Insufficient margin')
        }
        break
      case EOrderStatus.CANCELLED:
        console.log('Order cancelled')
        break
    }
  }
})
```
